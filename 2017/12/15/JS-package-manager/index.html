包管理javascript库，并提供读取和打包它们的工具

npm

npm官网

npm是node.js的包管理器。能够安装你想要的包，而且提供一个强大接口来使用它们。node.js让javascript编写服务器端应用程序成为可能。它建立在javascript V8（c++编写的）引擎之上，所以它很快。最初，它旨在为应用程序提供服务器环境，但是开发人员开始利用它来创建工具，帮助他们本地的任务自动化。此后，一个全新基于node工具（Grunt和Gulp）的生态系统，使得前端开发改头换面

  • 允许用户从npm服务器下载别人编写的第三方包到本地使用
  • 允许用户从npm服务器下载并安装别人编写的命令行程序到本地使用
  • 允许用户将自己编写的包或命令行程序上传到npm供别人使用

Node包模块

npm 有两种模式安装包，local(本地)和global(全球)。在local模式下，它会安装node_modules文件夹（在工作目录的父目录）中的包。global包安装在root用户下。这就意味需要使用sudo命令来安装global包，否则当解析第三方包依赖时，出于安全考虑可能会出现权限错误。

卸载local包

npm是一个包管理器，所以它肯定也可以卸载一个包。让我们假设当前版本的underscore包，存在兼容问题，移除当前的包然后安装一个老版本，如下：

$ npm uninstall underscore
-underscore@1.8.3 node_modules/underscore
$ npm list
/home/sitepoint/project
-(empty)

安装一个指定版本的包

$ npm install underscore@1.8.2

更新包

$ npm update underscore

管理缓存

当npm安装一个包的时候它会维护一个副本,下次你想再次安装那个包的时候，就不用再去访问网络重新下载了。那些副本会在home目录.npm文件夹中缓存起来。

$ ls ~/.npm

随时间推移，这个目录里会有很多老的包，十分混乱，所以时常清理下是十分必要的。

$ npm cache clean

管理依赖

随着包的数量的增长，手动安装依赖是非常不明智的，所以在项目中用package.json文件来管理它们，通过npm init 来创建

$ npm init
